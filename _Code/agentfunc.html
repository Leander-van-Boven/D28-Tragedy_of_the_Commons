Proposal Agent resource function

Take, for some simulation SSS, the set of all epochs TST_STS​ and the set of all agents ASA_SAS​. We want to define a function fff, such that:

f:TS×AS↦[0,1]f : T_S \times A_S \mapsto [0,1] f:TS​×AS​↦[0,1]

In other words, fff denotes the fraction of the available energy an agent aS∈ASa_S\in A_SaS​∈AS​ will take, based on the epoch tS∈TSt_S\in T_StS​∈TS​. The maximum amount of energy an agent will take is a pre-defined parameter ϵmax∈N\epsilon_{max}\in\mathbb{N}ϵmax​∈N. However, we need to make sure this amount of energy is available in the current epoch. We denote this value as ϵtS∈N\epsilon_{t_S}\in\mathbb{N}ϵtS​​∈N. The true amount of energy an agent will take at a certain epoch can then be described by the function E:TS×AS↦NE : T_S \times A_S \mapsto \mathbb{N}E:TS​×AS​↦N such that:

E(ts,as)=min⁡(f(ts,as)⋅ϵmax,ϵtS)E(t_s, a_s) = \min(f(t_s, a_s) \cdot \epsilon_{max}, \epsilon_{t_S}) E(ts​,as​)=min(f(ts​,as​)⋅ϵmax​,ϵtS​​)

We only need to define fff. The output of this function is influenced by different behaviours that an agent can emit. The agent acts on the current state of the simulation, and a behaviour defines how to act. However, multiple behaviours exist and these can have conflicting outcomes. In our simulation, the influence a behaviour can have on the outcome of fff depends on the agent's social value orientation, or svoa∈[0,1]svo_a\in [0,1]svoa​∈[0,1]. The total collection of nnn behaviours can be defined as a tuple

β=⟨B,M⟩,\beta ={\langle B, \Mu\rangle, } β=⟨B,M⟩,

Where BBB is an orderedset of nnn behaviour functions bi∈B:TS×AS↦[0,1]b_i\in B : T_S \times A_S \mapsto [0,1]bi​∈B:TS​×AS​↦[0,1], that 'vote' for the fraction of energy they 'advice' the agent to take. M\MuM is an ordered set of values μi∈M{\mu_i\in \Mu}μi​∈M that defines the 'optimal' social value orientation value for function bib_ibi​. First, we need to get for each behaviour, its distance from its optimal svo. Thus we define

D={d(μ,svoa)∣μ∈M}D = \{d(\mu, svo_a) | \mu \in \Mu\} D={d(μ,svoa​)∣μ∈M}

For some pre-determined distance function, such as: absolute, quadratic, etc. We can now define fff as

f(tS,aS)=combine(B(tS,aS),D))f(t_S, a_S) = \text{combine}(B(t_S, a_S), D)) f(tS​,aS​)=combine(B(tS​,aS​),D))

Where B(tS,aS)={b(tS,aS)∣b∈B}B(t_S, a_S) = \{b(t_S, a_S)|b\in B\}B(tS​,aS​)={b(tS​,aS​)∣b∈B}, and combine:[0,1]×M↦[0,1]\text{combine} : [0,1]\times M \mapsto [0,1]combine:[0,1]×M↦[0,1] is a function that combines all the behaviours based on the values in M\MuM. This can be done in multiple ways, for instance by taking the weighted average of BBB with 1−D1-D1−D as weights, or by taking the behaviour with the smallest distance.

The behaviours we've currently implemented can each be converted into a b∈Bb\in Bb∈B with a corresponding μ∈M\mu\in\Muμ∈M. Aport from this making the impacts continuous instead of discrete, I also think this will scale very well if we want to add behaviours later on. Additionally, this will make the behaviours not mutually exclusive, which I think is more realistic and will result in more complex and interesting results.
